# 测试覆盖率命令

分析测试覆盖率并生成缺失的测试。

## 工作流程

1. **运行带有覆盖率的测试**
   ```bash
   npm test --coverage
   # 或
   pnpm test --coverage
   ```

2. **分析覆盖率报告**
   - 读取 `coverage/coverage-summary.json`
   - 识别覆盖率 <80% 的文件

3. **生成测试**
   - 为未覆盖的函数生成单元测试
   - 为 API 生成集成测试
   - 为关键流程生成 E2E 测试

4. **验证**
   - 运行新测试
   - 检查通过
   - 显示改进的覆盖率

## 使用

```bash
/test-coverage
```

命令将：
- 运行覆盖率报告
- 识别低覆盖率文件
- 生成测试
- 显示改进指标
