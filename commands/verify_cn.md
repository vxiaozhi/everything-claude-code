# 验证命令

对当前代码库状态运行全面验证。

## 验证步骤

1. **构建检查**
   - 运行项目的构建命令
   - 如果失败，报告错误并停止

2. **类型检查**
   - 运行 TypeScript/类型检查器
   - 报告所有错误，文件:行

3. **Lint 检查**
   - 运行 linter
   - 报告警告和错误

4. **测试套件**
   - 运行所有测试
   - 报告通过/失败计数
   - 报告覆盖率百分比

5. **Console.log 审计**
   - 在源文件中搜索 console.log
   - 报告位置

6. **Git 状态**
   - 显示未提交的更改
   - 显示自上次提交以来修改的文件

## 使用

```bash
/verify
```

## 输出

生成简洁的验证报告：

```
VERIFICATION: [PASS/FAIL]

构建：    [OK/FAIL]
类型：    [OK/X 错误]
Lint：     [OK/X 问题]
测试：    [X/Y 通过，Z% 覆盖率]
机密：  [OK/X 发现]
日志：     [OK/X console.logs]

准备好 PR：[YES/NO]
```

## 参数

- `quick` - 仅构建 + 类型
- `full` - 所有检查（默认）
- `pre-commit` - 与提交相关的检查
- `pre-pr` - 完整检查 + 安全扫描
