# 代码审查命令

对最近的代码更改进行全面审查。

## 工作流程

1. **分析更改**
   ```bash
   git diff
   git status
   ```

2. **使用 code-reviewer 代理**
   - 审查安全性（机密、注入、XSS）
   - 检查代码质量（函数大小、复杂性）
   - 验证最佳实践（不可变性、错误处理）
   - 评估性能（算法、优化）

3. **生成报告**
   - 按优先级排序的问题（关键、高、中、低）
   - 具体的修复建议
   - 代码示例

## 使用

```bash
/code-review
```

## 审查类别

- **安全性**：OWASP 前 10 大、机密管理
- **质量**：代码复杂性、可读性
- **性能**：算法、优化
- **测试**：覆盖率、测试质量
- **文档**：注释、JSDoc
