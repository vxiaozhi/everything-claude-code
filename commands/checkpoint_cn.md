# 检查点命令

保存当前状态作为验证基线。

## 用途

1. **功能完成** - 在完成功能后保存检查点
2. **预提交** - 在提交前创建状态快照
3. **里程碑** - 标记重要的开发阶段

## 与 /verify 配合使用

```bash
/checkpoint              # 保存基线
# 进行更改...
/verify                  # 与检查点比较
```

## 检查点内容

- 构建状态
- 类型检查结果
- 测试结果
- 覆盖率百分比
- 依赖版本
- git commit SHA

## 持续评估

检查点是 **评估驱动开发** 中的 "基线" 阶段。
- 定义基线（检查点）
- 实施更改
- 运行验证
- 与基线比较
- 检测回归
