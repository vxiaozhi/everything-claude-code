# 编排命令

通过并行运行多个代理来协调复杂任务。

## 编配策略

### 1. 独立任务 - 并行
```markdown
启动 3 个代理并行：
1. 代理 1：ts/ 代码的安全性分析
2. 代理 2：components/ 的性能审查
3. 代理 3：utils.ts 的类型检查
```

### 2. 依赖任务 - 顺序
```markdown
运行代理序列：
1. 规划者代理 - 创建计划
2. ... 等待计划完成 ...
3. 架构师代理 - 审查架构
4. ... 等待审查完成 ...
5. 代码审查者代理 - 审查实施
```

### 3. 多视角分析 - 分角色
```markdown
从多个视角分析代码：
- 事实审查者
- 高级工程师
- 安全专家
- 一致性审查者
- 冗余检查者
```

## 使用

```bash
/orchestrate
```

提供任务描述，命令将：
- 分解为子任务
- 识别依赖关系
- 并行启动代理
- 聚合结果
