# 计划命令

为功能创建详细的实施计划。

## 规划流程

1. **需求分析**
   - 完全理解功能
   - 提出澄清问题
   - 识别成功标准

2. **架构审查**
   - 分析现有代码库
   - 识别受影响的组件
   - 审查类似的实现

3. **步骤分解**
   - 创建详细的步骤
   - 文件路径和位置
   - 依赖关系
   - 估计的复杂度

4. **实施顺序**
   - 按依赖关系优先级排序
   - 分组相关更改
   - 启用增量测试

## 使用

```bash
/plan 添加用户认证
```

代理将：
- 分析需求
- 检查现有代码库
- 创建详细计划
- 识别风险
- 建议实施顺序
