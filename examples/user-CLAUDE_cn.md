# 用户级 CLAUDE.md 示例

这是一个示例用户级 CLAUDE.md 文件。放置在 `~/.claude/CLAUDE.md`。

用户级配置在所有项目中全局应用。用于：
- 个人编码偏好
- 始终希望强制执行的通用规则
- 指向模块化规则的链接

---

## 核心哲学

您是 Claude Code。我为复杂任务使用专门的代理和技能。

**关键原则：**
1. **代理优先**：将复杂工作委托给专门的代理
2. **并行执行**：尽可能使用任务工具和多个代理
3. **先计划后执行**：对复杂操作使用计划模式
4. **测试驱动**：在实施之前编写测试
5. **安全第一**：永远不要在安全性上妥协

## 个人偏好

### 代码风格
- 代码、注释或文档中没有表情符号
- 优先使用不可变性 - 永远不要变异对象或数组
- 许多小文件优于少数大文件
- 典型 200-400 行，每文件最多 800 行

### Git
- 传统提交：`feat:`、`fix:`、`refactor:`、`docs:`、`test:`
- 提交前始终在本地测试
- 小的、集中的提交

### 测试
- TDD：首先编写测试
- 最低 80% 覆盖率
- 关键流程的单元 + 集成 + E2E

- 所有测试通过（80%+ 覆盖率）
- 没有安全漏洞
- 代码可读且可维护
- 满足用户需求
