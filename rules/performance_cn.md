# 性能优化

## 模型选择策略

**Haiku 4.5**（90% Sonnet 能力，节省 3 倍成本）：
- 频繁调用的轻量级代理
- 配对编程和代码生成
- 多代理系统中的工作代理

**Sonnet 4.5**（最佳编码模型）：
- 主要开发工作
- 编排多代理工作流
- 复杂的编码任务

**Opus 4.5**（最深的推理）：
- 复杂的架构决策
- 最大推理要求
- 研究和分析任务

## 上下文窗口管理

对于以下情况避免使用上下文窗口的最后 20%：
- 大规模重构
- 跨多个文件的功能实施
- 调试复杂的交互

较低的上下文敏感性任务：
- 单文件编辑
- 独立工具创建
- 文档更新
- 简单的错误修复

## Ultrathink + 计划模式

对于需要深度推理的复杂任务：
1. 使用 `ultrathink` 进行增强思考
2. 启用 **计划模式** 以进行结构化方法
3. 通过多轮批评"引擎"
4. 使用分角色子代理进行多样化分析

## 构建故障排除

如果构建失败：
1. 使用 **build-error-resolver** 代理
2. 分析错误消息
3. 增量修复
4. 每次修复后验证
