# 测试要求

## 最低测试覆盖率：80%

测试类型（全部必需）：
1. **单元测试** - 个别函数、工具、组件
2. **集成测试** - API 端点、数据库操作
3. **E2E 测试** - 关键用户流程（Playwright）

## 测试驱动开发

强制性工作流：
1. 首先编写测试（RED）
2. 运行测试 - 它应该失败
3. 编写最小实施（GREEN）
4. 运行测试 - 它应该通过
5. 重构（IMPROVE）
6. 验证覆盖率（80%+）

## 测试故障排除

1. 使用 **tdd-guide** 代理
2. 检查测试隔离
3. 验证模拟是正确的
4. 修复实施，而不是测试（除非测试是错误的）

## 代理支持

- **tdd-guide** - 主动用于新功能，强制首先编写测试
- **e2e-runner** - Playwright E2E 测试专家
