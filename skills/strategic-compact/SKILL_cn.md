# 战略压缩技能

建议在您的工作流中的战略点进行手动 `/compact`，而不是依赖任意的自动压缩。

## 为什么战略压缩？

自动压缩在任意点触发：
- 通常在任务中间，丢失重要的上下文
- 没有逻辑任务边界的意识
- 可能会中断复杂的多步骤操作

战略压缩在逻辑边界：
- **探索后，执行前** - 压缩研究上下文，保留实施计划
- **完成里程碑后** - 为下一阶段重新开始
- **主要上下文转换前** - 在不同任务之前清除探索上下文

## 工作原理

`suggest-compact.sh` 脚本在 PreToolUse（编辑/写入）上运行，并：

1. **跟踪工具调用** - 计算会话中的工具调用次数
2. **阈值检测** - 在可配置阈值处建议（默认：50 次调用）
3. **定期提醒** - 在阈值后每 25 次调用提醒

## 钩子设置

添加到您的 `~/.claude/settings.json`：

```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "tool == \"Edit\" || tool == \"Write\"",
      "hooks": [{
        "type": "command",
        "command": "~/.claude/skills/strategic-compact/suggest-compact.sh"
      }]
    }]
  }
}
```

## 最佳实践

1. **规划后压缩** - 计划最终确定后，压缩以重新开始
2. **调试后压缩** - 在继续之前清除错误解决上下文
3. **不要在实施中途压缩** - 为相关更改保留上下文
4. **阅读建议** - 钩子告诉您*何时*，您决定*是否*
